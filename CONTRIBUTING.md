# Contributing

We welcome contributions to Project Builder! Please follow these guidelines when contributing:

## Reporting Bugs

If you find a bug, please check if it has already been reported. If not, open a new issue with a clear description, steps to reproduce, and relevant environment details (OS, Go version).

## Feature Requests

If you have a feature request, please open an issue with a detailed explanation of the proposed functionality and its benefits.

## Pull Requests

1. Fork the repository.
2. Create a new branch for your feature or fix.
3. Make your changes and ensure they are well-commented.
4. Run tests: `go test ./...`
5. Commit your changes with clear, concise messages.
6. Push your branch to your fork.
7. Open a pull request against the main repository.

## Code Style

Project Builder follows standard Go formatting. Please use `go fmt ./...` before committing your changes.
